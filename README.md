# MTProto double-hop для Telegram (РФ → заграница)

Прокси для Telegram с двойным хопом. Клиент подключается к серверу в РФ, оттуда трафик уходит по обфусцированному туннелю AmneziaWG на заграничный сервер, и уже он ходит к Telegram. Снаружи всё выглядит как обычный TLS на непопулярный порт.

```
клиент → A (РФ, HAProxy) → AmneziaWG → B (заграница, telemt) → Telegram
```

Зачем так:

- **Входной IP (РФ)** — это то, что видит клиент и что попадает в ссылку `tg://proxy?...`. Российский IP меньше привлекает внимание и быстрее отвечает.
- **Выходной IP (заграница)** — это то, что видит Telegram. Реальный заграничный сервер не светится клиентам напрямую.
- Между ними — туннель AmneziaWG с рандомизированными параметрами обфускации (Jc/Jmin/Jmax/S1/S2/H1–H4), у каждой установки свои.
- На выходе `telemt` отдаёт FakeTLS под видом обращения к какому-нибудь популярному домену (по умолчанию `www.microsoft.com`).

## Что нужно

- Два сервера: один в РФ (вход), один за границей (выход).
- Ubuntu 24.04 (на 22.04 тоже работает), root на обоих.
- На выходном сервере — выход в интернет к github (ставится `telemt` и PPA AmneziaWG).

## Установка

Скрипт один и тот же на обоих серверах, роль выбирается в меню. **Начинать надо с выходного сервера** — он печатает токен, который понадобится на входном.

### 1. Выходной сервер (заграница)

```bash
sudo -i
curl -fsSL https://raw.githubusercontent.com/trebore/-MTProto-RU-EU/main/telemt-doublehop.sh -o telemt-doublehop.sh
chmod +x telemt-doublehop.sh
./telemt-doublehop.sh
```

Выбираешь пункт `1) ВЫХОДНОЙ`. Скрипт спросит:

- публичный IP этого сервера (определяет сам, если не вышло — введёшь руками);
- публичный IP входного (РУ) сервера — он попадёт в ссылку;
- клиентский порт (по умолчанию 2053, можно поставить 443);
- UDP-порт туннеля;
- FakeTLS-домен.

Дальше он сам поставит AmneziaWG, поднимет туннель, поставит и настроит `telemt`, откроет нужные порты в ufw (если стоит) и в конце напечатает **ТОКЕН** — длинную base64-строку. Скопируй её целиком.

### 2. Входной сервер (РФ)

```bash
sudo -i
curl -fsSL https://raw.githubusercontent.com/trebore/-MTProto-RU-EU/main/telemt-doublehop.sh -o telemt-doublehop.sh
chmod +x telemt-doublehop.sh
./telemt-doublehop.sh
```

Выбираешь пункт `2) ВХОДНОЙ`, вставляешь токен с выходного сервера, жмёшь Enter. Скрипт поднимет свою сторону туннеля, поставит HAProxy, проверит, что выход пингуется, и выдаст готовую ссылку `tg://proxy?...` для Telegram.

> Ссылку, которую печатает родной инсталлятор `telemt` (с IP выходного сервера), использовать **не нужно** — она ведёт мимо двойного хопа.

## Управление

Повторный запуск скрипта на любом из серверов открывает админ-меню (роль определяется автоматически).

**На выходном (B):**

- показать ссылки всех пользователей;
- добавить / удалить пользователя;
- статус туннеля и число активных соединений;
- сменить FakeTLS-домен;
- полное удаление.

**На входном (A):**

- показать ссылку;
- статус и примерное число уникальных устройств;
- сменить клиентский порт;
- полное удаление.

## Как это устроено

| Компонент | Где | Роль |
|-----------|-----|------|
| HAProxy | вход (A) | принимает клиентов на клиентском порту, проксирует в туннель с `send-proxy-v2` |
| AmneziaWG | A ↔ B | обфусцированный туннель, подсеть `10.10.10.0/24` (A=.2, B=.1) |
| telemt | выход (B) | MTProto-прокси, слушает туннельный IP, FakeTLS, ходит к Telegram |

Несколько деталей, которые скрипт делает сам:

- `telemt` биндится на туннельный IP, поэтому через systemd drop-in он стартует **после** туннеля (иначе бинд падает на ребуте).
- Внутренний порт `telemt` наружу не торчит и в ufw открыт только из туннеля; если порт по умолчанию занят — подбирается следующий свободный.
- Обе пары ключей WireGuard генерируются на выходном сервере (оба сервера твои); приватный ключ входного уезжает внутри токена.
- Ключи и параметры обфускации хранятся в `/etc/amnezia/amneziawg/`, состояние установки — в `/etc/telemt-doublehop/params.env`.

## Файлы и пути

- `/etc/amnezia/amneziawg/awgN.conf` — конфиг туннеля
- `/etc/telemt/telemt.toml` — конфиг прокси (только на выходе)
- `/etc/haproxy/haproxy.cfg` — конфиг HAProxy (только на входе)
- `/etc/telemt-doublehop/params.env` — параметры установки

## Диагностика

- Туннель не поднимается на входе — чаще всего битый токен (вставился не целиком или с переносами строк). Скопируй заново.
- Пинг до выхода не проходит — проверь, что на выходном сервере открыт UDP-порт туннеля для IP входного.
- Статус соединений и свежесть handshake смотри в админ-меню, пункт «Статус».

## Удаление

Запусти скрипт → «Полное удаление». Сносится только то, что поставил он сам: туннель, telemt/HAProxy и состояние установки. Остальные сервисы и туннели на сервере не трогаются.
